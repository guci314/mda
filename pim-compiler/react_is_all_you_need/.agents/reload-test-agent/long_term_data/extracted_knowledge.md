# 知识库

## 元知识
- 通过用户明确指令获取关键信息的方法（直接记忆用户指定的重要数据）
- 通过用户提问验证记忆准确性的方法（当用户询问已记忆内容时进行确认）

## 原理与设计
- 重要数字信息可能具有象征意义（如42代表生命/宇宙的答案）
- 系统需保留用户明确要求记忆的核心数据
- 记忆验证机制需保持数据一致性（用户多次询问时返回相同结果）

## 接口与API
- 记忆指令格式："请记住[信息]，这是[描述]"
- 信息确认响应模板："已记住[信息]，这是[描述]。需要时可提醒"
- 记忆查询响应模板："你让我记住的数字是[信息]，这是[描述]！"

## 实现细节（需验证）
- 数字类信息的存储可能通过短期记忆缓存实现
- 记忆检索功能需保持数据持久性（跨对话保持）
- 注：实现细节可能已变化，使用前需验证

## 用户偏好与项目特点
- 该用户可能引用文化典故（如《银河系漫游指南》的42梗）
- 对具有哲学/文化含义的数字信息需要特殊标记
- 用户倾向于通过重复提问验证系统记忆能力
