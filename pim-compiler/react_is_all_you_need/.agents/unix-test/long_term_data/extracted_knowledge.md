# 知识库

## 元知识
- 知识程序可以通过参数匹配执行不同功能分支
- 执行流程遵循"解析参数→匹配逻辑→返回结果"的标准模式

## 原理与设计
- 知识程序采用"多分支响应"设计理念
- 核心概念：知识即程序，通过参数化执行体现
- 架构决策：基于字符串匹配的简单路由机制

## 接口与API
- 程序接口：输入任意字符串参数
- 输出模式：
  - 含"hello"→友好问候
  - 含"calculate"→执行计算
  - 纯数字→返回平方
  - 其他→返回大写形式

## 实现细节（需验证）
- 参数解析采用字符串包含检测（非精确匹配）
- 处理逻辑按优先级顺序执行（hello优先于数字检测）

## 用户偏好与项目特点
- 测试程序偏好简单直接的字符串处理
- 典型模式：条件分支→格式化输出
