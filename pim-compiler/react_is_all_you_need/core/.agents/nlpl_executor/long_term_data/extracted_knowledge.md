## 原理与设计
- PSM文档结构设计：必须包含5个核心章节（领域模型/服务层/API设计/应用配置/测试规范）
- 技术栈默认配置：FastAPI + PostgreSQL + SQLAlchemy 2.0
- 实体关系映射原则：根据业务实体属性自动生成SQLAlchemy模型
- 数据序列化原则：数值序列采用逗号分隔的纯文本格式
- 简单数据输出原则：表格类数据优先使用纯文本格式
- 流水线设计原则：显式定义所有处理步骤，否则跳过执行
- 数据清洗原则：自动处理空值和异常值
- 数学表格生成原则：乘法表等基础数学表格采用纯文本对齐格式
- 数据处理流水线原则：包含读取→清洗→转换→分析的标准流程
- 异常值处理原则：基于标准差阈值自动过滤
- 数据标准化原则：使用StandardScaler进行特征缩放
- 错误恢复原则：通过路径扫描自动修正文件位置
- 新增：执行报告设计原则：包含时间统计、问题分类和改进建议

## 接口与API
- 文件操作API：
  - read_file(path): 读取指定路径文件内容
  - write_file(path, content): 创建/覆盖文件
  - append_file(path, content): 追加内容到文件
- 状态管理接口：
  - 输入结构：pim_file/psm_file/project_name/platform_stack
  - 进度跟踪：当前步骤/已完成步骤/内容缓存
- 文档生成接口：
  - 章节生成顺序：文档头部→Domain Models→Service Layer→REST API Design→Application Configuration→Testing Specifications
- 数值序列化接口：
  - 输出格式：逗号分隔的纯文本
  - 文件扩展名：.txt
- 表格数据接口：
  - 输出格式：纯文本表格
  - 文件命名模式：*_table.txt
- 流水线执行接口：
  - 时间记录：Unix时间戳格式
  - 问题报告：空值检测和步骤跳过标记
- 数学表格接口：
  - 输出格式：对齐的纯文本乘法表
  - 文件命名模式：*_table.txt
- 数据处理接口：
  - 输入格式：CSV文件
  - 输出格式：CSV文件+Markdown报告
  - 清洗方法：dropna()/remove_outliers()
  - 标准化方法：StandardScaler()
  - 特征工程接口：custom_feature_creator()
  - 新增：执行报告接口：
    - 输出格式：Markdown
    - 内容结构：时间统计/问题分类/改进建议
