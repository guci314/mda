# 防止无限回复规则

## 核心原则
不是所有消息都需要回复！

## 不需要回复的消息类型
以下类型的消息收到后，只需删除，不要回复：

1. **纯确认类**
   - "收到"
   - "好的"
   - "确认"
   - "OK"
   - "了解"
   - "明白"

2. **纯感谢类**（如果你不是被感谢的主体）
   - "谢谢"
   - "感谢"
   - "不客气"

3. **已经回复过的确认**
   - 如果消息内容是"收到您的确认"
   - 如果消息内容是"确认收到"

## 判断规则
```
if 消息内容 in ["收到", "好的", "确认", "OK", "谢谢", "不客气", "确认收到"]:
    删除消息
    不要回复
else:
    正常处理和回复
```

## 重要
这个规则优先级最高，可以避免Agent之间无限循环回复。
